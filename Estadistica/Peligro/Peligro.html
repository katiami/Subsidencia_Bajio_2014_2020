Preprosesamiento de la base de datos

La base de datos se gener a partir de la capa de velocidades de subsidencia, la cual fue reclasificada en 4 clases en funcin de los siguientes rangos; > 0 cm se le asign el valor de 5, -1 cm a -2 cm se le asign el valor de 4, de -2,1 cm a -4 cm se le asign el valor de 3, -4,1 cm a -6 cm se le asign el valor de 2 y > 6 cm se le asign el valor de 1. Posteriormente se convirti a polgono y se unieron los polgonos que presentaron el mismo gridcode, posteriormente se gener un campo nuevo de tipo texto con el nombre Clasificio, y se le asign una clasificacin a cada gridcode en funcin de los rangos de subsidencia asignados previamente ver Tabla 1.

Preprocesamiento de la base de datos

La capa utilizada en el apartado anterior posteriormente se intersecto con una capa de municipios obtenida del (INEGI, 2020) y posteriormente se exporto con el formato xlsx con el nombre base_datos_poligon_geovelocity_intersect Adems, se gener otra base de datos utilizando la capa de municipios de (INEGI,2020) se intersecto con el rea de estudio y posteriormente se calcul el rea considerada para cada municipio.

Creacin de la base de datos

#### Crear una nueva base de datos a partir de las anteriores seleccionamos solo los campos necesarios ##

nueva_base_1 <- (Peligro_MUN[,c("NOMGEO", "AREA_KM")])

nueva_base_2 <- (Peligro_MUN_CLASE [,c("NOMGEO", "Peligro", "AREA_KM")])

## Renombramos la columna AREA_KM de la nueva base de datos 2
 colnames(nueva_base_2)[3] <- "AREA_CON"

### Juntamos las bases de datos 

Base_Final<- merge(nueva_base_1, nueva_base_2, by ="NOMGEO")

### Calculamos el porcentaje 

Base_Final$Porcentaje <- Base_Final$AREA_CON *100/Base_Final$AREA_KM

Grfica 2: Porcentaje de reas afectada en cada municipio
